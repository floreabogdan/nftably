Base chains are usually named after their hook: input, forward, output, prerouting, postrouting.
The fields below only apply to base chains. A regular chain ignores them.
input = traffic addressed to this box · output = traffic this box sends · forward = traffic routed through it · prerouting/postrouting = for NAT (rewriting addresses as packets arrive / leave) · ingress/egress = earliest/latest, bound to one interface.
The interface this ingress/egress chain binds to — these hooks require one (they run per-interface, before the rest of the pipeline).
filter for allow/deny rules · nat for port-forwards and masquerade · route for policy routing.
Order among chains on the same hook. Keywords work: filter, dstnat (prerouting NAT), srcnat (postrouting NAT), or a number.
The verdict for traffic that reaches the end of the chain unmatched. drop is the secure choice for an input chain — but then you must add rules to allow the traffic you want (including your own SSH / this UI), or the auto-revert will have to save you.
