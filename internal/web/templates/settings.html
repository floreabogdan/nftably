Expose a Prometheus /metrics endpoint so your firewall can be graphed and alerted on in Grafana/Prometheus. Every rule with a Count action becomes a time series (nftably_rule_packets_total, nftably_rule_bytes_total), alongside table/chain/rule counts, an nftably_up health gauge, and build info.

{{if .Settings.MetricsToken}}
Endpoint enabled on
Scrapers must send this as Authorization: Bearer <token>. Keep it secret; treat it like a password.

Example Prometheus scrape config:

scrape_configs:
  - job_name: nftably
    scheme: http
    authorization:
      credentials: {{.Settings.MetricsToken}}
    static_configs:
      - targets: ['{{if .Settings.ListenAddr}}{{.Settings.ListenAddr}}{{else}}127.0.0.1:8080{{end}}']

Note: /metrics is still behind the access list — add your Prometheus server's IP there if it scrapes from another host.

{{else}}
Endpoint disabled off

Off by default. Enabling mints a random bearer token; the endpoint returns 404 until then, and rejects any request without the token.

Download your whole configuration — every table, chain and rule, plus the named sets and their entries — as one portable JSON file. It's the model, not the database: no login credentials, no settings, so it's safe to share, keep in version control, or move to another box.

{{template "icon-alert"}}Restore configuration

Load a backup file. This replaces your entire current configuration (all tables, chains, rules and named sets) with the file's contents. It's model-only — you land on Changes to review the diff and apply behind the armed auto-revert, so nothing reaches the kernel until you confirm. Export your current config first if you might want it back.
