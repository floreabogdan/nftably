{{template "icon-route"}}Inspect transit with an IDS/IPS {{if .IDSAdded}}added · off{{end}}

Hand forwarded traffic to an inline intrusion detector — Suricata or Snort running on an NFQUEUE — so it can flag or block known-bad flows. It's fail-open: if the inspector isn't running, traffic keeps flowing, so a stopped Suricata never blackholes your transit. Only the forward chain is touched — your own session is never queued.

{{template "icon-route"}} Attach an IDS/IPS to NFQUEUE 0

nftably's rule sends forwarded packets to queue num 0. Point one inspector at that queue and it inspects the traffic inline. The rule is fail-open, so start the inspector before you rely on it to block — while nothing is attached, traffic passes uninspected. The kernel needs the nfnetlink_queue module (usually auto-loaded on first use).

Suricata (inline IPS)

Install it, make sure the ruleset uses drop actions (that's what makes it an IPS, not just an IDS), then run it inline on queue 0:

sudo apt install suricata            # Debian/Ubuntu

# run inline on NFQUEUE 0 (foreground, to try it)
sudo suricata -q 0

# as a service: add "-q 0" to the run options and enable
# IPS mode, e.g. in /etc/default/suricata or the systemd unit.
sudo systemctl enable --now suricata

A source dropped by Suricata never reaches the rest of your forward chain. Alerts land in /var/log/suricata/.

Snort 3 (inline via the nfq DAQ)
# Snort 3 reading NFQUEUE 0 in inline mode (-Q)
sudo snort -c /etc/snort/snort.lua \
  -Q --daq nfq --daq-var queue=0 -s 65535

Snort 2.x is similar: snort -Q --daq nfq --daq-var queue=0 -c /etc/snort/snort.conf.

Want a different queue number or fail-closed behaviour? Edit the Send to userspace program action on the Firewall page.
